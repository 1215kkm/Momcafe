# 동네맘 — 컨셉 문서

> 동네 단위로 *매일 하나의 결정*을 같은 동네 엄마들과 *익명·동시*에 내리는 일일 의식(ritual) 앱.
> 시범 동네: **강서구** / 타깃: **3~7세 유아 엄마**

---

## 1. 진단 — 왜 기존 맘카페는 게시판에 머물렀나

네이버 맘카페가 단순 게시판에서 진화 못 한 진짜 이유는 *기술 부족이 아니라 정체성 오진*이다.

- **표면 정체성**: "정보 게시판" — 검색·태그·인기글로 강화하려고 함
- **진짜 본질**: "내 결정을 검증받고 싶은 익명의 약자들의 합의 인프라"
- **모순**: 본질을 건드릴수록 *익명 vs 신뢰* 문제에 부딪혀, 진화하려고 할 때마다 본질을 죽이는 쪽(실명 인증·등급제·운영진 검수)으로 갔음
- **구조 한계**: 카페 = 영주제 → 글 쓰는 소수만 발언권, 나머지는 눈팅. 30~40대 엄마는 이미 다른 곳에서도 발언권이 약한 사람들

**결론**: 본질을 *그대로 둔 채* 익명·신뢰·발언권 세 문제를 푸는 새 구조가 필요하다.

## 2. 30~40대 엄마의 숨은 결핍 5가지

| 결핍 | 정체 | 강도 |
|---|---|---|
| 사라진 1인칭 | "○○ 엄마"로만 호명됨. 내 이름·취향·일과가 가족 시스템에 흡수 | 압도적 |
| 판단 검증 결핍 | 매일 수십 개 결정(이유식·학원·접종·남편 행동)을 내려야 하는데 검증할 곳이 없음 | 매우 강 |
| 시간 빈곤·인지 과부하 | 5분 단위 멀티태스킹, 깊게 생각할 여력 없음 | 매우 강 |
| 죄책감 화폐 | "내가 부족한 엄마인가" — 모든 결정에 죄책감이 따라붙음 | 강 |
| 수평 동료 부재 | 같은 처지 친구를 새로 사귀기 너무 어려움 | 강 |

## 3. 핵심 심리 레버 — psychology.md에 없는 두 개의 신규 레버

표준 12레버 외에 이 타깃에 *특히 강한* 신규 레버 둘을 도입한다.

### 레버 A. 대리 결정 / 책임 분산 (Decision Outsourcing)
- **자극**: "내가 결정하기 무서운 걸 집단의 판단으로 옮긴다"
- **행동 유도**: 매일의 결정을 *집단 응답*과 함께 내림 → 죄책감·실패 책임이 분산됨
- **환금**: 결정 데이터 자체가 자산이 된다 (같은 고민을 한 N명의 통계 = 동네 의사결정 인프라)

### 레버 B. 정체성 회복 (Identity Restoration)
- **자극**: "○○맘이 아니라 *나*로 호명되는 1분"
- **행동 유도**: 결혼 전 이름·직업·전공으로 호명되는 짧은 순간 → 다른 어디서도 못 얻는 가치
- **환금**: 강력한 락인. 구독·프리미엄 결제 정당화

### 매일 들어오게 만드는 조합
**대리 결정(A) + 정체성 회복(B) + 호기심 갭(#11) + 인정(#2)** 4중 조합.

- A → "오늘 내가 내려야 할 결정, 다른 엄마들은?" 검색하러 옴
- B → 검색 끝나도 *나로 머무는 1분*이 있어서 안 떠남
- #11 → 어제 던진 결정이 24h 뒤 어떤 응답으로 돌아왔나 보러 다시 옴
- #2 → 응답이 모이면 다른 사람 결정에 내가 끼어들고 싶어짐 → 글 쓰는 사람으로 전환

## 4. 제품 정의 — "동네맘"

### 한 줄 정의
> **맘카페를 *게시판*이 아니라 "오늘 내가 내려야 할 결정 1개"를 같은 동네 엄마 100명과 *익명·동시* 결정하는 일일 의식 앱으로 재정의.**

### 메인 기능 — "오늘의 결정 (Decision of the Day)"

**(1) 매일 아침 7시 — "오늘의 결정" 푸시**
앱이 지난 24시간 동안 *같은 동네 엄마들이 가장 많이 검색·고민*한 결정 1개를 자동 추출해서 푸시.
> 예: "오늘 강서구 7세 엄마들의 결정 — 이번 주말 키즈카페 vs 도서관, 당신은?"

**(2) 30초 응답 — A/B + 한 줄 이유**
2초가 아니라 30초. A/B 선택 + 한 줄 이유 입력. **입력 전에는 다른 사람 결과를 못 본다 (커튼)** — 사회적 동조 편향 차단.

**(3) 응답 직후 — 1분 정체성 회복 화면**
응답하면 바로:
> "오늘 OOO님의 선택은 A입니다. 예전 직업: 디자이너. 같은 직업이었던 다른 엄마 12명도 A를 골랐어요."

"○○맘"이 아니라 *예전의 나*로 호명. 가입 시점에 *결혼 전 직업·전공·취미*를 받아둠 (이게 모방 불가 자산).

**(4) 24시간 후 — "어제의 결정 결과" 푸시**
> "강서구 키즈카페 선택자 47명 중 31명이 '만족', 16명이 '후회'. 후회 이유 1위: ○○."

어제 내 결정의 *결과 데이터*가 콘텐츠. 매일 다시 오는 이유.

**(5) 비동기 토론 (제한적)**
응답에 단 한 줄 이유에 *공감 / 다시 생각* 두 버튼. **댓글은 공감 받은 사람만 달 수 있게** — 신입 발언권 0 문제를 푼다. 한 줄 이유만 잘 쓰면 인정받을 수 있음.

---

### 보조 기능 4-α — "이름 회복실 (Identity Room)"

정체성 회복 레버를 *독립 기능*으로 분리.

- 주 1회 30분, 익명 음성/텍스트 라운지
- *결혼 전 이름·전공·꿈*으로만 자기소개
- **규칙 단 하나: 자녀·남편·시댁 얘기 금지**
- 신청 → 같은 동네·같은 옛 직업/전공 4명 매칭 → 30분 후 자동 종료

> 셋로그의 시간 동기성을 빌리되, 인증이 아닌 *정체성 회복*이라는 다른 본질에 붙임.

---

### 보조 기능 4-β — "내일의 나에게 추천 (Forward Recommendation)"

- 오늘의 결정에서 *후회한 사람*이 *내일 같은 결정을 할 사람*에게 메모를 남기는 구조
- 예: "강서구 7세 엄마, 내일 같은 고민할 사람에게 — 키즈카페 가지 마. 차라리 ○○ 가세요."
- **시간 차 호혜성 (#7)** — 어제의 내가 내일의 누군가에게 빚을 갚는다
- 부수 효과: 어제 글이 오늘의 데이터가 됨 → 광고 없이 콘텐츠 자생

## 5. 수익 경로 — 광고도 단순 구독도 아닌 4가지

### (1) 결정 데이터 → 동네 가게 B2B 리서치 자산화 [메인 줄기]
> "강서구 7세 엄마 100명이 이번 주말 도서관을 선택했고 47%가 후회"
> = 키즈카페 사장에게 5만원짜리 리포트.

광고를 *받지 않고* 데이터를 *판다*. 본질이 *동네 의사결정 인프라*가 됨.

### (2) "후회율" = 자연 발생 가게 평점 (간접 환금)
별점·후기 시스템 없이 *후회율 데이터*가 자연히 가게 평가가 됨. **유저는 후기 0개 작성, 평점은 만들어진다.**

### (3) 정체성 회복 = 프리미엄 락인 (LTV)
"이름 회복실" 무료 주 1회. 무제한·1:1 매칭 **월 9,900원**. 핵심: 해약 시 잃는 것이 명확함 → *매주 한 시간 나로 사는 시간*. 손실 회피(#3) × 정체성.

### (4) 데이터의 외부 자산화
1년치 *지역별·연령별 47만 건 결정 데이터*는 육아 콘텐츠·교육 회사·지자체에게 *광고가 아닌 리서치 자산*. 통계청·소비자 패널이 못 가진 *지역·시간 해상도*. 단가는 광고의 10배 이상.

> **핵심 분기**: 광고를 안 팔고 *의사결정 데이터를 판다*. 1·2·3안과 가장 큰 차이.

## 6. 선제 방어

| 예상 역공 | 방어 |
|---|---|
| 익명 데이터 판매 → 사생활 침해 논란 | 개인 식별값 절대 안 받음. 최소 응답 50명 집계에서만 데이터화. 가입 시 *호혜성 프레임*으로 동의 ("내 결정이 동네 의사결정에 도움이 됩니다") |
| 네이버·당근이 베끼기 | *예전 직업·전공* 정체성 데이터는 모방 불가 자산 (가입자 명부엔 없음). 6개월 누적이 해자. "이름 회복실" 상표/서비스명 등록 |
| "답변 강요 느낌"으로 이탈 | 푸시 1일 1회 고정 7시. 응답률 낮은 사용자는 *질문 난이도 낮춤* (한 줄 이유 → A/B 추측). 댓글권은 *공감 받은 한 줄 이유*가 보상 |

## 7. 시스템화 단계 (수동 → 도구)

1. **사람이 매일 "오늘의 결정" 질문을 손으로 큐레이션** — 강서구 시범 1곳, 약 3개월
2. 누적 검색·답글 키워드에서 **질문 자동 생성기**
3. "어제의 결정 → 오늘의 결과" 자동 회수 봇 — 24h 뒤 응답자에게 만족도 1탭 푸시
4. B2B 대시보드를 *동네 사장 SaaS*로 분리 — 본 앱은 데이터 생산 기지, 매출은 SaaS에서
5. 데이터셋 규모 (동네 100곳 × 1년) 도달 시 *지자체·교육청·소비자 패널 라이센스* → 본 앱은 무료여도 손해 안 보는 구조

## 8. 메인 안과 보조 안의 관계

```
   [메인]                  [보조 4-α]               [보조 4-β]
오늘의 결정         →    이름 회복실         →     내일의 나에게
(매일 7시)              (주 1회 30분)              (어제→내일 메모)

매일 들어오는 입구       정체성 회복 자체를         후회한 사람이 내일
DAU 엔진                심화하는 프리미엄          유저에게 가치 전달
                       락인 (월 결제)              호혜성 → 재방문 루프
```

세 계층이 서로를 완성한다:
- **메인**이 매일 들어올 이유를 만든다 (대리 결정)
- **4-α**가 *나로 머무는 시간*을 깊게 만든다 (정체성 회복, 프리미엄 락인)
- **4-β**가 *어제·내일을 잇는다* (호혜성, 콘텐츠 자생)

## 9. 적용한 원칙·레버·수익 요약

- **적용한 아뱅 원칙**: #1 근본 원인 우선 / #5 심리 욕구 중심 설계 / #6 프레임 재정의(게시판→의식) / #4 손이 막히면 시스템으로 / #3 선제 방어 / #9 수익 직결
- **당긴 심리 레버**: 신규 A 대리 결정 / 신규 B 정체성 회복 / #11 호기심 갭 / #2 인정 / #3 손실 회피 / #7 호혜성
- **수익 경로**: 의사결정 데이터의 B2B 자산화 + 정체성 프리미엄 락인(LTV) + 외부 데이터 라이센스 — 광고·단순 구독 둘 다 아님
