# 06. 피벗 — 단일 앱 → 멀티 카페 플랫폼

> 2026-05-21 사용자(CEO) 결정.
> 4번째 안의 "오늘의 결정" 메커니즘은 폐기하지 않는다 — *플랫폼 위의 옵션 모듈*로 흡수한다.

---

## 1. 무엇이 바뀌나

| 축 | Before (01~05 문서) | After (이 피벗) |
|---|---|---|
| 정체성 | 단일 동네(강서구) 일일 의식 앱 | **누구나 카페를 만드는 멀티 모임 플랫폼** |
| 입구 | 매일 7시 "오늘의 결정" 푸시 | **디스커버리 첫 화면** — 필터로 모임을 발견 |
| 단위 | 1개 동네·1개 결정 | **N개 모임 × N개 게시판** |
| 운영 | 우리가 큐레이션 | **모임 운영자(유저)가 자유 생성** |
| 4번째 안 메커니즘 | 메인 기능 | **카페별로 켤 수 있는 옵션 모듈** ("오늘의 결정"·"이름 회복실"·"내일의 나에게") |

## 2. 헌법 5원칙 재정의 (필수)

기존 헌법은 *단일 동네 의식 앱*을 전제. 멀티 플랫폼에서는 일부 원칙 수정.

| # | 원래 | 수정안 |
|---|---|---|
| P1 | "게시판이 아니다" | **플랫폼은 게시판 *위에* 있다.** 카페 안엔 게시판이 있을 수 있고, 그 위에 *심리 모듈*(오늘의 결정 등)을 카페가 켤 수 있다. 플랫폼 자체는 *모임을 발견·생성·운영*하는 레이어. |
| P2 | "익명 + 정체성 회복 동시" | **카페별 옵션.** 정체성 회복은 일부 카페(엄마·이직자·전공 동문 등)에서만 의미. *기본은 닉네임*, 카페가 정체성 모듈을 켜면 결혼 전 이름·전공 표기. |
| P3 | "응답 전 결과 비공개 (커튼)" | **"오늘의 결정" 모듈 켜진 카페에만 적용** — 일반 게시판엔 해당 없음 |
| P4 | "댓글권은 보상" | **모듈별 옵션.** 일반 게시판은 자유 댓글, 심리 모듈은 댓글권 메커니즘. |
| P5 | "수익은 광고 아님" | **유지.** *모임 데이터의 B2B 자산화* + *플랫폼 수익(유료 모임·프리미엄 운영자 도구)*. 광고 인벤토리 X. |

상세는 본 문서 받은 뒤 `.claude/knowledge/momcafe/project-context.md`를 AI팀이 패치할 것.

## 3. 사용자가 명시한 첫 화면 사양 (요구사항)

### A. 필터
- 지역별
- 인기별 (참여가 활발한 순)
- 회원수별

### B. 내 모임 섹션
- 내가 만든 모임
- 내가 가입된 모임
- 카드 형식 노출

### C. 모임 카드 (1줄 1모임)
- 좌측: 로고
- 우측: **최근글 2개 + 인기글 2개**, 위아래로 배치
- 카드만 보고도 *흥미로운 내용*이 보여야 함 (=클릭 유도)

### D. 모임 생성 + 게시판
- 누구나 모임 생성 가능
- 모임 운영자가 *필요할 때마다* 게시판을 추가할 수 있음
- 게시판 종류·아이디어 게시판 기능은 **AI팀이 제안**

## 4. AI팀에 던지는 질문

### → 아뱅 (아이디어뱅크)
1. 멀티 카페 플랫폼의 *심리적 차별점*은 뭔가? (네이버 카페·당근 모임과 어떻게 다른가)
2. 누가 *카페를 만들고 싶어지는가* — 그 동기에 어떤 심리 레버를 거는가?
3. 첫 화면에 추가할 만한 *통념을 깬* 아이디어 1~2개 (사용자가 "또 아이디어있으면 알려줘"라고 명시)
4. **아이디어 게시판이란 게 뭐여야 하는가** — 일반 게시판과 어떻게 다른가, 어떤 심리 메커니즘이 작동하는가
5. 모임 안에 켤 수 있는 *특수 게시판 시리즈* 5~7개 제안 (각각의 심리 레버 + 수익 경로)
6. 기존 4번째 안(오늘의 결정·이름 회복실·내일의 나에게)이 *모듈*로 들어갈 때 어떻게 작동해야 하나

### → UX 디자이너
1. 첫 화면 IA — 디스커버리/내 모임/검색/생성 진입점
2. 모임 카드 레이아웃 (1줄, 로고 + 최근글 2 + 인기글 2) — 360px 모바일에서 어떻게 자연스럽게 표시
3. 필터 UI (지역·인기·회원수) — 한꺼번에 / 토글 / 다중선택?
4. 모임 생성 플로우
5. 모임 내부 IA (게시판 추가·관리·멤버 관리·심리 모듈 토글)
6. 게시판 생성 플로우 (운영자가 어디서 어떻게 새 게시판 만드는지)
7. 기존 32개 화면(S01~S43)과의 관계 — 살릴 것·옵션 모듈로 변환할 것·폐기할 것

### → UI 디자이너 (UX 끝난 뒤)
- 디스커버리 카드 와이어 (모임 카드)
- 새 IA에 맞는 화면 와이어 추가
- 기존 디자인 시스템(03)이 멀티 카페 톤에 맞는지 검토 — 색은 *모임마다 다르게* 표시될 텐데 시스템 톤은 유지

## 5. 클로드의 첫 화면 추가 아이디어 (아뱅 호출 전 미리 던지는 씨앗)

사용자에게 우선 보여주고, 아뱅이 *더 강한 안*을 낼 거다.

| 안 | 무엇 | 어디에 |
|---|---|---|
| **오늘의 모임 추천 1장** | 알고리즘이 *나에게 맞을 만한 모임 1개*를 매일 1장 카드로. 검색 부담 0, 호기심 갭(#11) | 디스커버리 최상단 |
| **방금 핫한 글 스트림** | 모임을 가로지르는 *지금 이 순간 가장 빠르게 댓글 달리는 글* 5개. FOMO(#4) | 카드 리스트 위 또는 사이에 끼우는 가로 스크롤 |
| **내가 비슷한 사람들이 가입한 모임** | 사회적 증거(#5) — "예전 디자이너였던 분 12명이 가입한 모임" | 내 모임 섹션 하단 |
| **임시 핫 토픽 태그** | "오늘 이 키워드로 글이 47개 올라왔어요" — 카페 가로지르는 키워드 발견 | 검색 진입점 근처 |
| **모임 만들기 진입을 카드처럼** | "이 동네에는 [걷기 모임]이 없어요. 첫 운영자가 되어보세요" — 결핍을 직접 자극 | 디스커버리 끝부분 |
