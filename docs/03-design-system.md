# 동네맘 — 디자인 시스템

> 본 문서는 `./.claude/knowledge/ui-designer/`에 별도 오버라이드가 없는 상태에서,
> 동네맘 프로젝트의 *시각 언어*를 직접 정의한다.
> 글로벌 Crowny 시스템(보라/핑크 그라데이션)은 *맘카페·여성앱 클리셰*에 가까워
> 헌법(P4 게시판 아님 / P6 ○○맘 호명 금지 / 1일 1결정 의식)과 충돌한다.
> 따라서 동네맘은 별도의 톤을 채택하되, 토큰 구조·간격 그리드는 글로벌과 호환되게 둔다.

## 0. 디자인 원칙 (모든 시각 결정의 기준)

| # | 원칙 | 시각 변환 |
|---|---|---|
| V1 | **의식(ritual)이지 SNS가 아니다** | 흰 여백, 침묵, 1화면 1행동. 피드형 카드 쌓기 금지. |
| V2 | **맘카페 클리셰 회피** | 파스텔 핑크·하트·곰돌이·리본 *전부 금지*. 통념 깨기. |
| V3 | **시간 빈곤 → 큰 글자, 큰 탭 영역** | 본문 17px 기본, 버튼 56px 높이, 좌우 24px 패딩. |
| V4 | **상태가 톤을 바꾼다** | 응답 전(커튼)·정체성 카드(회복)·결과(공개)·후회(따뜻함) 각각 다른 *분위기*. |
| V5 | **호명은 시각의 중심이다** | `{결혼전이름}` `{직업/전공}`은 *가장 큰 글자*. ○○맘 어디에도 없음. |
| V6 | **장식은 위계가 있을 때만** | 그라데이션·아이콘은 *오늘의 결정의 핵심 순간*에만. |

## 1. 색 팔레트

### 1-1. 메인 — 이끼 그린(Moss) & 점토(Clay)

맘카페 통념(핑크/파스텔/라일락)을 *의식적으로 피한다*. 대신 **이끼 그린**을 메인으로 채택.

| 토큰 | HEX | 왜 |
|---|---|---|
| `--color-primary` | `#3D5A47` | 짙은 이끼 그린. 차분·안정·*결정의 무게*. 핑크 클리셰 회피. |
| `--color-primary-soft` | `#7A9885` | 옅은 이끼. 보조 액션. |
| `--color-primary-bg` | `#EEF2EC` | 배경 톤. 카드 바닥. |
| `--color-accent` | `#C36B3C` | 점토 오렌지. 강조 1개 — *정체성 카드의 호명 라인*에만. |
| `--color-accent-soft` | `#E8C9A8` | 점토 라이트. 후회 메모 카드 배경. |

> *근거*: 그린은 "검증된 결정"의 색이자, 핑크의 정반대에 있는 *조용한 신뢰*의 톤. 점토 오렌지는
> 정체성 회복 순간을 시각적으로 *유일하게* 강조하기 위한 강조색 1개 — 다른 데 쓰지 않는다.

### 1-2. 상태색 (시스템)

| 토큰 | HEX | 용도 |
|---|---|---|
| `--color-success` | `#4A7C59` | 만족 응답, 완료 상태 |
| `--color-regret` | `#A65A4A` | 후회 응답 (빨강 아님 — *판단을 비난하지 않는* 적색) |
| `--color-warning` | `#C99A3D` | 매칭 대기, 시간 임박 |
| `--color-danger` | `#8B3A2E` | 삭제·탈퇴 (위험 액션) |

> *근거*: 빨강(`#E11D48`)을 *쓰지 않는다*. 후회를 잘못된 결정처럼 비추지 않기 위해 톤다운된 적갈색.

### 1-3. 중립 (텍스트·배경)

| 토큰 | HEX | 용도 |
|---|---|---|
| `--text-primary` | `#1C1C1C` | 본문 — 거의 검정. 가독성 1순위. |
| `--text-body` | `#3A3A3A` | 일반 텍스트 |
| `--text-muted` | `#6B6B6B` | 보조 정보, 캡션 |
| `--text-faint` | `#A0A0A0` | placeholder, 비활성 |
| `--bg-canvas` | `#FAFAF7` | 앱 전체 배경 — 순백 아닌 *종이톤* |
| `--bg-surface` | `#FFFFFF` | 카드·시트 |
| `--bg-elevated` | `#F4F4EF` | 인풋·서브 카드 |
| `--border-soft` | `#E5E2DA` | 카드 테두리 |
| `--border-strong` | `#C7C2B5` | 입력 테두리 |

### 1-4. 응답 전 "커튼" 색 (P3 강제)

응답 전 화면(S10)은 *통계가 가려진 상태*. 시각적으로도 *덮인 톤*.

| 토큰 | HEX | 용도 |
|---|---|---|
| `--curtain-bg` | `#2D3A33` | S10 상단 배경 — 짙은 이끼 (커튼이 내려와있는 느낌) |
| `--curtain-text` | `#F4F4EF` | 커튼 위 흰 글자 |
| `--curtain-divider` | `#4A5F52` | 커튼 안 구획 |

> 응답하면 커튼이 *걷히듯* `--bg-canvas`로 전환 (애니메이션 §7 참조).

### 1-5. 정체성 카드 색 (S12 — 제품의 심장)

| 토큰 | HEX | 용도 |
|---|---|---|
| `--identity-bg` | `linear-gradient(170deg, #F4ECD8 0%, #E8C9A8 100%)` | 카드 배경 — *따뜻한 종이*. |
| `--identity-name` | `#C36B3C` | {결혼전이름} 라인. 점토 오렌지. |
| `--identity-job` | `#3D5A47` | {직업/전공} 라인. 메인 그린. |
| `--identity-body` | `#3A3A3A` | 본문 텍스트 |

> *근거*: 정체성 카드는 *오래된 편지*의 느낌. 그라데이션·네온 X. 종이결의 따뜻한 베이지.

### 1-6. 절대 쓰지 않을 색 (금기)

- 파스텔 핑크 `#FFC0CB`, 라일락 `#C8B6E2`, 코랄 핫 핑크 `#FF6F91`
- 형광 그린, 비비드 옐로우
- 보라 그라데이션 (글로벌 Crowny 톤 — 본 프로젝트에선 *오버라이드*)
- 순수 빨강 `#FF0000`, `#E11D48` (후회·삭제도 적갈색 사용)

## 2. 타이포그래피

### 2-1. 폰트

```
기본:     'Pretendard', 'Noto Sans KR', -apple-system, sans-serif
정체성:   'Noto Serif KR' (S12 정체성 카드의 호명 라인 1군데만 — 편지 느낌)
아이콘:   'Material Symbols Rounded' (라인, weight 400)
숫자(통계): 'Pretendard' (Tabular Nums 활성)
```

> *근거*: Pretendard는 한국어 본문 가독성 표준. S12의 호명 라인 1군데만 Serif로 *질감 차이*를
> 줘서 다른 어디서도 못 보는 순간임을 시각으로 신호한다.

### 2-2. 크기·굵기 체계

| 토큰 | 크기 / weight / line-height | 용도 |
|---|---|---|
| `--ts-display` | 32px / 800 / 1.2 | S12 정체성 카드 호명 라인 (Serif) |
| `--ts-title` | 24px / 700 / 1.3 | 페이지 타이틀 (S10 질문문) |
| `--ts-heading` | 20px / 700 / 1.35 | 섹션 제목 |
| `--ts-body-lg` | 17px / 500 / 1.55 | 본문 기본 — *30~40대 가독성 베이스* |
| `--ts-body` | 15px / 400 / 1.55 | 일반 텍스트 |
| `--ts-caption` | 13px / 500 / 1.4 | 보조 정보 |
| `--ts-tag` | 12px / 600 / 1.2 | 칩·배지 |
| `--ts-nav` | 11px / 600 / 1.2 | 바텀 네비 라벨 |

### 2-3. 타이포 사용 규칙

- **한 화면 최대 3단계 위계**. 그 이상은 *시각 위계 실패*.
- **본문은 17px이 기본**. 14px 이하는 *경고/푸시 부속 정보 외*에는 쓰지 않는다.
- **호명 `{결혼전이름}`은 본문 안에서도 +2px 굵기 700**. 글자 자체가 *상태 시각*.
- **A/B 버튼 라벨은 22px / 700**. 30초 응답이라 한눈에 읽혀야 함.

### 2-4. 글자 크기 접근성

`--fs` CSS 변수로 -2 ~ +4 단계. 본문 17px 기준 ±2px 단위. *Settings(S43)에서 사용자 조절*.

## 6. 컴포넌트 명세

### 6-1. 버튼

#### 주 버튼 (Primary)
- 높이 56px, 패딩 `--sp-5` 좌우, 둥글기 `--r-md`
- 배경 `--color-primary` (`#3D5A47`)
- 글자 흰색 / `--ts-body-lg` / 700
- 상태:
  - hover/press: 배경 `#2F4738` + `--shadow-press`
  - disabled: `#C7C2B5` 배경 / `#FAFAF7` 글자
  - loading: 글자 자리에 16px 스피너 (이끼색) + "잠시만요"

#### 보조 버튼 (Secondary, 아웃라인)
- 높이 48px, 둥글기 `--r-md`
- 배경 투명, 테두리 `1.5px solid --color-primary`, 글자 `--color-primary`
- press: 배경 `--color-primary-bg`

#### A/B 선택 카드 버튼 (S10 전용)
- 큰 카드 형 버튼. 높이 120px, 둥글기 `--r-lg`
- 배경 `--bg-surface`, 테두리 `1px solid --border-soft`
- 라벨 22px / 700, 부제 14px / 400 / `--text-muted`
- press 시: 테두리 `2px solid --color-primary` + 배경 `--color-primary-bg`
- 선택 후: 0.6s 후 S11로 전환 (커튼이 위로 걷히는 모션)

#### 위험 버튼 (Danger — 탈퇴 등)
- 텍스트 버튼만. 글자 `--color-danger`, 밑줄 없음, `--ts-body` / 500
- press: 배경 `rgba(139, 58, 46, 0.08)`

#### 공감/다시 생각 버튼 (S13 인라인)
- 칩 형. 높이 36px, 둥글기 `--r-pill`
- 공감: 배경 `--color-primary-bg`, 글자 `--color-primary`, 아이콘 `favorite` (라인)
- 다시 생각: 배경 `--bg-elevated`, 글자 `--text-body`, 아이콘 `psychology_alt`
- *이름 변경 금지*: "좋아요" 아닌 "공감", "싫어요" 아닌 "다시 생각"

### 6-2. 입력 필드

#### 일반 인풋 (S04 이름, S05 직업)
- 높이 52px, 둥글기 `--r-sm`
- 배경 `--bg-elevated`, 테두리 `1px solid --border-strong`
- placeholder `--text-faint` / 17px
- 포커스: 테두리 `2px solid --color-primary`, 배경 `--bg-surface`
- 에러: 테두리 `2px solid --color-regret`, 하단 13px 메시지

#### 한 줄 이유 인풋 (S11 — 30자)
- 둥글기 `--r-md`, 패딩 16px 18px
- 우측 하단에 글자수 카운터 `0 / 30` — 13px / `--text-muted`

#### 메모 인풋 (S16 — 60자)

### 6-3. 카드

#### 한 줄 이유 카드 (S13 피드)
- 둥글기 `--r-lg`, 패딩 `--sp-5`, 배경 `--bg-surface`, `--shadow-card`
- 상단: `{직업/전공}` 칩 (예: "디자이너였던 분") — 12px / 600 / `--color-primary`
- 본문: 한 줄 이유 17px / 500
- 하단: 공감/다시 생각 버튼 + 공감 수 (`favorite` 아이콘 + 숫자)
- "내가 쓴 한 줄"은 상단 핀: 테두리 `1.5px solid --color-primary-soft` + 상단 라벨 "내가 쓴 한 줄"

#### 정체성 카드 (S12)
- 풀스크린. 배경 `--identity-bg` (베이지 그라데이션)
- 카드 자체 둥글기 없음 (풀스크린이므로)
- 상단 32px 여백 후 30초 카운트다운 점 4개 (작게)
- 중앙:
  - 줄1: "오늘" — 17px / 400 / `--text-muted`
  - 줄2: `{결혼전이름}` 님의 선택은 — 32px / Serif / 700 / `--identity-name`
  - 줄3: `[A]` — 28px / 700 / `--text-primary` (선택 결과)
  - 줄4: 여백 32px
  - 줄5: 예전 `{직업/전공}`이셨던 다른 엄마 — 17px / 500
  - 줄6: **12명**도 오늘 `[A]`를 골랐어요 — 22px / 700 / `--identity-job`
  - 줄7: 여백 48px
  - 줄8: 마이크로 카피 — 15px / 400 / Serif italic / `--text-muted`
    예: "디자이너였던 당신, 오늘도 안목이 통했네요."
- 하단: 60초 진행 바 (얇은 1px 라인, `--color-primary-soft` → 0%로 줄어듦)
- 우상단: [닫기] 텍스트 버튼 (회색)

#### 메모 카드 (S13 하단 — "어제의 누군가")
- 둥글기 `--r-lg`, 배경 `--color-accent-soft` (점토 라이트)
- 상단: "어제 같은 결정을 한 디자이너였던 엄마의 메모" — 13px / 500 / `--text-muted`
- 본문: 60자 메모 — 17px / 500 / `--text-primary`
- 하단 우측: [고마워요] 버튼 (작은 칩, 둥글기 pill, 14px / 600, `--color-primary-bg` 배경)
- 댓글창 *없음* (P4 게시판화 방지)

#### 결과 막대 (S13 상단 통계)
- 가로 막대. 높이 28px, 둥글기 `--r-sm`
- 왼쪽 A: 폭 = A 비율 %, 배경 `--color-primary`
- 오른쪽 B: 폭 = B 비율 %, 배경 `--color-primary-soft`

- 응답 50명 미만이면 막대 회색조 (`--bg-elevated`) + "더 모이면 알려드려요" 캡션

### 7-4. 24h 후 — 후회·만족 톤 (S15)

## 10. 애니메이션 / 트랜지션

| 종류 | 정의 |
|---|---|
| 페이지 전환 | `fadeIn 0.25s ease` + 6px translateY |
| 커튼 걷힘 (S10→S11) | 상단 짙은 영역이 위로 슬라이드, 0.6s `cubic-bezier(0.4, 0, 0.2, 1)` |
| 정체성 카드 입장 | 베이지 배경 페이드 0.3s → 호명 라인 페이드 0.4s 0.3s delay |
| 60초 카운트다운 바 | linear, width 100→0%, 60s |
| 결과 막대 채워짐 | 0→실제 비율, 0.8s `ease-out` |
| 토스트 | 하단에서 12px 슬라이드 + 페이드, 2.5s 후 사라짐 |
| A/B 카드 선택 | 테두리 1px→2px (0.15s) + 배경 페이드 |

> *과한 모션 금지*. 인지 과부하 타깃. 0.6s 넘어가는 모션은 *정체성 카드 입장* 한 곳만.

## 11. 접근성 & 반응형 베이스라인

- **최소 본문 17px** (30~40대 시력 고려)
- **터치 영역 최소 44x44**
- **명도 대비**: 본문 `#1C1C1C` on `#FAFAF7` = 16:1 (AAA)
- **컬러 외 신호**: 후회/만족은 색만이 아니라 *아이콘*과 *라벨*도 동반
- **다크 모드**: v1 미지원. 단 토큰 구조는 호환되게 둠.
- **모바일 우선**: 360px 폭 기준. 최대 폭 420px.
