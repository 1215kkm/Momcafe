## Q1. 네이버 카페·당근 모임과의 *심리적 차별점*

### 숨은 진짜 문제
네이버 카페·당근의 본질은 *공간 제공자*다. "방을 줄게, 알아서 채워라." 모임이 만들어진 뒤엔 관여 안 한다. 결과: 카페 99%가 *생성 → 며칠 활성 → 죽음* 사이클로 끝나고, 시체 카페가 산처럼 쌓여 있다.

- 운영자는 *왜 내 카페만 죽는지* 모른다
- 가입자는 *어느 카페가 진짜 살아있는지* 모른다 (회원수만 보고 들어가면 죽은 카페)

**진짜 막힌 지점**: 네이버 카페는 *카페의 생존*에 무관심. 우리가 끼어들 자리.

### 안 1. 모임에 "심박수" 박기 (Heartbeat)
- 회원수·게시글수 같은 *누적 지표*를 첫 화면에서 지움. 대신 **지난 24시간 심박수** 하나만 표시
- 심박수 = (24h 작성글 × 2) + 댓글 + 모듈 응답 + 신규가입. 죽으면 0 + 카드 회색
- 네이버 카페가 *과거의 영광*(회원 30만)으로 가입자를 속이는 구조 부숨

### 안 2. "사망 임박" 알림 (Death Warning)
- 심박수 7일 연속 하락 시 운영자에게 **"이 모임은 14일 뒤 사망 위험. 응급처치 모듈을 켜시겠어요?"** 푸시
- 네이버 카페가 절대 안 하는 짓: 운영자의 무능을 가시화

### 안 3. 장례식 + 흡수 모델
- 사망 선고된 카페의 게시글·회원 명단을 비슷한 주제 활성 카페가 *흡수* 가능
- 원 운영자 자존감 보존, 새 운영자엔 씨앗 멤버 무료 공급

### 수익
CAC↓ (가짜 카페에 안 낚임) + 운영 SaaS B2B + 장례식 흡수 수수료

### 적용 원칙·레버

## Q2. 카페를 만들고 싶어지는 *동기 설계*

### 숨은 진짜 문제
네이버·당근의 카페 생성 동력은 *영주 판타지*("내 왕국") → 왕좌에 오른 뒤 *권태로 방치* → 시체 더미. 같은 동력 쓰면 같은 결과. **다른 동기**가 필요.

### 4가지 결핍 자극 안

| 안 | 핵심 | 심리 레버 |
|---|---|---|
| **검색 결과 0 폭격** | "이 키워드로 검색한 사람 7일간 53명. 첫 운영자가 되시겠어요? 이미 53명이 대기 중" | #5 사회적 증거 + #3 손실회피 |
| **"이름 회복 카페"** | 가입 시 받은 *예전 직업·전공* 활용. "디자이너였던 30명이 강서구에 있어요. 디자이너 출신 엄마 모임 운영자가 되시겠어요?" | 신규 B 정체성 회복 + #8 자기효능감 |
| **지도 결핍** | "강남구 7개, 송파구 4개, 마포구 3개 — 강서구 0개" 역방향 사회적 증거 | #5 사회적 증거(역방향) + #3 손실회피 + #1 경쟁 |
| **씨앗 멤버 5명 보장** | 운영자가 첫 글 1개만 쓰면 알고리즘이 *비슷한 관심사 유저 5명에게 초대 푸시* 무료. 0→1 마찰을 우리가 떠안음 | #7 호혜성 + #9 매몰비용 |

### 수익
공급자 비용 0 / 유저→유저 유입 / 데이터 축적

## Q3. 첫 화면에 추가할 *통념 깬* 안

씨앗 5개(추천·발견 기반)는 네이버·당근도 한다. **비틀기 축**: 유저를 *받는 사람* → *판정자·기여자·당사자*로 옮기기.

### 안 1 (강력 추천). "오늘의 모임 결정" — 4번째 안 메커니즘을 *플랫폼 레벨*에서 매일 가동
씨앗 안 "오늘의 모임 추천 1장"을 *뒤집음*. 알고리즘이 추천 → **모든 유저가 하루 하나의 모임을 함께 판정.**

- 매일 19시 첫 화면 최상단에 *한 모임* 클로즈업 (최근 7일 심박 폭증 모임 또는 특이 컨셉 신생 모임)
- 30초 안에 **A: 가입 / B: 보류** — 응답 전 결과 비공개 (커튼, P3)
- 응답 후 *"강서구 30대 디자이너 출신 47명이 가입"* 노출
- 익일 아침 — *"어제 가입한 87명 중 23명이 이미 첫 글을 썼어요"* 결과 푸시

**통념 깸 지점**: 알고리즘 추천 → 집단 큐레이션. 필터 버블 우회. 4번째 안 메커니즘이 *플랫폼 입구 DAU 엔진*이 됨.

### 안 2. "방금 사라진 모임" 카드 — 역방향 FOMO
씨앗 안 "방금 핫한 글"을 뒤집음. 모두가 *지금 뜨는 것*을 보여줄 때 우리는 **방금 죽은 것**.

- "오늘 사망 선고를 받은 모임 3개"
- 마지막 글·마지막 글 작성자 노출. "이 사람은 외로웠을까?"
- 클릭 → **장례식 모드** → 흡수 신청 (Q1 안3 입구)
- 동시에 유사 활성 모임 추천 (이동 유도)

## Q4. "아이디어 게시판"의 정체 — 모임의 두뇌

### 일반 건의함이 죽는 3가지 함정
1. 제안자만 외로움 (답이 안 옴)
2. 운영자의 폭정 (안 채택하면 끝)
3. 책임 회피 (누가 해?)

### 4단계 메커니즘

| 단계 | 핵심 |
|---|---|
| **1. 제안 (Propose)** | 제목 + 한 줄 요약 + 예상 효과 1줄 + *내가 맡을 부분* 1줄(선택). 맡은 사람 있는 아이디어 우대 노출 |
| **2. 양방향 투표 (Bi-vote)** | 찬성/반대 + *한 줄 이유 필수*. 반대표가 더 가치 있음. 응답 전 결과 비공개 |
| **3. 자동 채택 (Auto-Adopt)** | *운영자 승인 없이* 활성 회원 30%+ 찬성 + 반대 한줄 동의자 < 반대표의 50% 시 자동 공식 안건 승급. 운영자는 *거부권만*, 거부 사유 공개 의무 |
| **4. 실행 추적 (Track)** | 진행률 14일 0% 시 자동 재공모. 완료 아이디어는 모임 프로필에 *영구 기록* |

### 통념 깸
운영자 권한을 알고리즘으로 제한 → 운영자 폭정 해제.

### 수익
운영 SaaS(4단계 대시보드 유료) + 거버넌스 데이터 외부 판매

## Q6. 4번째 안 *모듈화* — 운영자 처방

모듈은 *기능 리스트*가 아니라 **심박 회복 처방**으로 포지셔닝.

| 모듈 | 운영자 진단 | 약속 결과 | 한 줄 카피 |
|---|---|---|---|
| **오늘의 결정 모듈** | "멤버가 글 안 씀, 눈팅만, 운영자 혼자 떠듦" | "응답률 60% 이상이면 심박 2주 안에 3배" | *눈팅 90%를 응답자 60%로 바꾸는 모듈* |
| **이름 회복실 모듈** | "멤버끼리 데면데면, 새 멤버 안 들어옴" | "주 1회 30분 정체성 만남으로 친밀도 폭증, 첫 글 쓰는 사람 4배" | *처음 보는 사이를 동료로 만드는 모듈* |
| **내일의 나에게 모듈** | "어제 좋았던 글이 오늘 묻힘, 신규 멤버가 과거 글 안 봄" | "콘텐츠 수명 7배, 운영자 새 글 안 써도 됨" | *콘텐츠가 자생하는 모듈* |

### 수익

## 종합 — 이 안 전체가 만드는 차별

> 네이버 카페·당근이 **공간 임대업**이면 우리는 **모임 생물학**이다.

- 가입자는 *살아있는 심박만 본다*
- 죽어가는 운영자는 *처방을 받는다*
- 만들고 싶은 사람은 *결핍 자극 + 씨앗 멤버*로 만든다
- 카페가 죽으면 *장례식과 흡수*
- 모든 모임이 *옵션 모듈*을 켤 수 있다
- 모든 모임이 *아이디어 게시판*으로 운영자 폭정을 푼다

## 수익 경로 5가지
1. 모임 운영 SaaS (B2B 운영자 도구) — 심박·진단·처방
2. 정체성 회복 프리미엄 (B2C 월 9,900원)
3. 집단 결정 데이터 자산화 (B2B 외부 판매)
4. 장례식 흡수 모델 (죽은 카페 환금)
5. 모임 거버넌스 SaaS (아이디어 게시판 4단계 추적 B2B)
