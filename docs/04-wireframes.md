# 동네맘 — 와이어프레임 (32 화면)

> 본 문서는 `02-ux-flow.md`의 화면 ID 인덱스와 *필수/금지/주의*를 시각 골격으로 변환한 것이다.
> 시각 토큰은 `03-design-system.md`. 픽셀이 아니라 *토큰 이름과 상태별 동작*으로 핸드오프.
> 모바일 360px 폭 기준. ASCII 박스는 모노스페이스 환경에서 정렬됨.

## 범례

```
┏ ━ ┓   화면 상단/하단 경계
│        세로 경계
[버튼]   탭 가능한 영역
{변수}   동적 텍스트 (헌법 토큰)
···      여백 / 침묵
※        주의 사항
```

## UI 작업 중 발견된 *UX 디자이너 재확인 필요* 항목

1. **S15 만족/후회 응답의 *재진입 가능 여부*** — "한 번만 응답"이라고 와이어에 적었으나 02-ux-flow.md엔 명시되어 있지 않다. *수정 1회 허용 vs 잠금*?
2. **S30 댓글권 누적 한도** — 공감 3건당 1장이라면 *동시 보유 최대*는? 1장만 보유 가능인지, 누적 보유 가능인지 명시 필요.
3. **S25 라운지 4명 미만 *진행 중* 이탈 처리** — 30분 안에 한 명이 강제 종료/이탈하면 3명으로 계속 진행하는지, 자동 종료하는지 룰 미정.
4. **S40 결정 히스토리 표시 기간** — "최근 14일"로 잡았으나 UX 문서엔 기간 미정. 14일 / 30일 / 전체?
5. **S13 한 줄 이유 카드 최대 노출 수** — "최대 20개"로 와이어에 적었으나 UX 문서 미명시. 게시판화 방지 관점에서 *상한*은 필요하나 정확한 숫자 합의 필요.
6. **E1 사용자의 S13 결과 피드 노출 정책** — 정체성 데이터 미입력자가 S13에서 "예전 {직업}이셨던 분" 카드들을 볼 때 *직업 호명*은 정상 노출되는지, 일반화되는지?
