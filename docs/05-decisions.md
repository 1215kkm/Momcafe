# 열린 결정 — 1차 마감

UI 디자이너가 와이어 작성 중 발견한 6개의 모호한 지점에 대한 **1차 결정**. 02-ux-flow.md / 04-wireframes.md의 해당 화면 명세는 이 결정을 따른다. 향후 시범 운영 데이터로 재검토 가능.

판단 기준:
- 헌법 5원칙 (게시판 아님 / 익명 + 정체성 회복 / 커튼 / 댓글권은 보상 / 광고 아님)
- 30~40대 엄마의 5결핍 (사라진 1인칭 / 판단 검증 결핍 / 시간 빈곤 / 죄책감 화폐 / 수평 동료 부재)
- 합리적 단순함 우선 (시범 단계에서 규칙이 복잡할수록 신뢰 깨짐)

---

## D1 — S15 만족/후회 재응답 가능 여부

**결정**: **응답 후 24h 내 1회 변경 가능. 그 후 락.**

근거:
- "결정한 직후 vs 24h 살아본 후"는 다른 감각이라 *최초 1회 수정*은 데이터 정합성에 오히려 도움
- 그러나 무제한 변경은 *후회율 통계 = 가게 평점*의 기반을 깬다 (수익 경로 (2) 직격)
- 24h 락은 시간 빈곤 타깃에 강요 아닌 *경계*로 작용

UI에 미치는 영향:
- S15 만족/후회 토글에 *변경 가능 시간 카운터* 1줄 캡션 (예: "23시간 17분 후 잠겨요")
- 락 이후엔 토글 비활성화 + 회색 처리

---

## D2 — 댓글권 동시 보유 최대치

**결정**: **동시에 1장만 보유. 누적 불가. 24h 미사용 시 소멸.**

근거:
- 헌법 4: 댓글권은 *보상*이지 *권력*이 아니다. 누적되면 신입↔고인물 격차 = 등급제 부활
- 1장 + 소멸 = 손실 회피(#3) × 단순함. "지금 안 쓰면 사라진다"는 압력이 곧 *댓글의 진정성* 확보
- 운영 복잡도 0 — 시범 단계에 필수

UI 영향:
- S30/S31의 티켓 카드는 *한 장만* 표시
- 이미 1장 보유 중인 사용자가 새 공감을 받으면 → "다음 티켓은 현재 티켓 사용 또는 만료 후 발급" 안내

---

## D3 — S25 라운지 진행 중 이탈자 처리

**결정**:
- 4명 중 1명 이탈 → 3명 계속 진행
- 잔류 2명 이하 → 5분 카운트다운 후 자동 종료
- 자동 종료된 잔류자에게 **다음 회복실 *우선 매칭권* 1장** 자동 지급

근거:
- 정체성 회복은 *기대했던 시간을 잃는 것*에 가깝다 — 손실 회피 강하게 작동
- 우선 매칭권은 호혜성(#7) × 호기심 갭(#11) — "다음 회는 어떤 사람과 매칭될까"
- 시범 단계에 *실시간 보충 매칭*은 비현실적 → 단순 카운트다운 + 우선권이 최선

UI 영향:
- S25에 *남은 인원 표시* (4 / 3 / 2 — 2명 이하 진입 시 카운트다운 5:00 시작)
- 종료 화면 S26 변형: "오늘 30분이 너무 짧게 끝났네요. 다음 회 우선 매칭권을 드릴게요."

---

## D4 — S40 히스토리 표시 기간

**결정**: **최근 30일. 그 이전은 "월별 요약 카드"로 압축.**

근거:
- 시간 빈곤 타깃에 무한 스크롤은 함정. 30일이 *한 달 단위 회고* 감각과 맞음
- 월별 요약은 *후회율·만족율·내가 남긴 메모 수*만 표시 → 콘텐츠 자생 + 자기 정체성 누적의 동시 효과
- 30일 = 시범 운영 단위(3개월)에도 자연스러움

UI 영향:
- S40 메인은 30일 카드 리스트
- 스크롤 끝나면 "이전 달 요약 보기" 진입점 — 가벼운 1탭

---

## D5 — S13 한 줄 이유 카드 노출 상한

**결정**: **TOP 5 (공감순) + "더 보기" 진입 시 최대 20개.**

근거:
- 헌법 1: 게시판이 되면 안 됨. 노출 무제한은 게시판화의 첫 신호
- TOP 5는 *훑는 시간 30초 이내*에 맞음 (시간 빈곤)
- 20개 상한은 신입에게도 *공감 받을 기회*를 보장 (게시판 1페이지는 보통 50~100개 → 신입 묻힘)
- 5 위 → 20 위까지 노출은 *내가 쓴 한 줄도 보일 수 있다*는 기대를 유지하는 임계

UI 영향:
- S13에서 카드 5장 노출 후 "다른 분들의 한 줄도 보기 (15개)" 진입점
- 더 보기 화면도 *공감순* 정렬 고정 — 최신순 정렬 옵션 제공 금지 (게시판화 차단)

---

## D6 — E1 정체성 미입력자가 보는 다른 사람의 호명

**결정**: **정상 노출. 단 S12-empty에서 부드러운 유도 1줄.**

근거:
- 다른 사람이 "예전 디자이너였던 분"으로 호명되는 걸 보는 순간 → *나도 입력하고 싶어진다* (호기심 + 호혜성 트리거)
- 강제 입력 게이트는 헌법 위반급 부담 — 30~40대 엄마 가입 시 *마찰 0*이 핵심
- S12-empty의 안내 문구 한 줄이면 유도 충분

UI 영향:
- S12-empty 화면에:
  > "당신의 예전 이름·직업을 알려주시면, 다음 결정부터 같은 길을 걸어온 분들 옆에 묶어드릴게요."
- 입력 진입 버튼은 *작게*, 무시 버튼은 *동등하게* — 강요 아님

## 결정에 묶인 UI/UX 변경 요약

| 결정 | UX 문서 (02) | UI 문서 (04) |
|---|---|---|
| D1 | S15 토글 수정 가능 시간 명시 | S15 캡션 + 락 후 비활성 상태 |
| D2 | 티켓 동시 1장 명시 | S30/S31 티켓 카드 1장 표시 |
| D3 | E3 이탈 분기 룰 명시 | S25 인원 표시 + S26 변형 |
| D4 | S40 30일+월별 요약 명시 | S40 카드 리스트 + 진입점 |
| D5 | S13 카드 상한 TOP 5 / 20 | S13 더 보기 진입점 + 정렬 고정 |
| D6 | E1 호명 정상 노출 + 유도 룰 | S12-empty 안내 문구 |

향후 02/04 문서를 본격 *코드용 명세*로 굳힐 때 위 표대로 패치한다 (이번 세션의 와이어/플로우는 이 결정의 *방향*까지만 반영. 라인 단위 갱신은 MVP 직전).

## 아직 안 닫힌 것 (다음 의사결정 라운드용)

- **푸시 알림 톤·문구 라이브러리** — 매일 7시·24h 후·티켓 만료 5분 전 등 (마케터 영역)
- **B2B 리포트 형식·가격표** — 가게당 5만원 가설값. 실제 시범 운영 시 인터뷰 필요
- **이름 회복실 음성/텍스트 비율** — 현재는 양쪽 다 가능. 시범 운영에서 *어느 쪽이 정체성 회복에 더 강한지* 측정 필요
- **개인정보 처리방침 실무** — 정체성 데이터 보유·삭제 규정 (보안 에이전트 검토 필요)
