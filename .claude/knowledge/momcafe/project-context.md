# 심박(구 동네맘) 프로젝트 컨텍스트

이 파일은 강팀(9명) 모두가 작업 시작 전 먼저 읽는 프로젝트 헌법이다.

---

## 제품 한 줄

> **심박** — 누구나 카페를 만드는 *멀티 모임 플랫폼*. 네이버 카페가 *공간 임대업*이면 우리는 **모임 생물학**이다. 카페의 *생사*를 책임진다.

## 시범 환경

- **지역**: 강서구 (Genesis 1호 모임으로 보존) + 자유 생성 허용
- **타깃**: 3~7세 유아 자녀를 둔 엄마 + 누구나
- **기간**: 3개월
- **응답풀 목표**: 첫 동네 100명, 플랫폼 전체 다양

## 절대 원칙 (피벗 후 갱신, 어기면 제품이 죽는다)

1. **플랫폼은 게시판 *위에* 있다.** 카페 안엔 게시판이 있고, 그 위에 *심리 모듈*을 카페가 켤 수 있다. 플랫폼 자체는 *모임을 발견·생성·운영*하는 레이어. *게시판 강화·검색·태그 단순 개선*은 답이 아니다.
2. **익명 + 정체성 회복은 카페별 옵션.** "○○맘" 호명 전역 금지(헌법 보존). 정체성 회복 모듈을 켠 카페만 결혼 전 직업·전공으로 호명. 기본은 닉네임.
3. **응답 전 결과 비공개 (커튼) — 결정 모듈에서만.** 일반 게시판엔 해당 없음. 사회적 동조 편향 차단.
4. **댓글권은 보상 — 심리 모듈에서.** 공감 받은 한 줄 이유에만 댓글권. 일반 게시판은 자유 댓글.
5. **수익은 광고가 아니다.** ① 운영 SaaS ② 정체성 프리미엄 ③ 데이터 자산화 ④ 장례식 흡수 ⑤ 거버넌스 SaaS 5종. 광고 인벤토리 0.

## 추가 원칙 (셋로그 통합 후)

6. **영상은 폰에 산다.** 셋로그 모듈의 영상은 *폰에만* 저장 + 메타만 서버. 폰 분실·교체 시 영상 사라짐이 *기능 약속*.

## 핵심 심리 레버

- **신규 A. 대리 결정 (Decision Outsourcing)** — 결정 책임을 집단으로 분산
- **신규 B. 정체성 회복 (Identity Restoration)** — 예전의 나로 호명되는 1분
- 표준 12레버 중 #2 인정, #3 손실회피, #7 호혜성, #11 호기심 갭 조합

## 메인 + 보조 안 구조

```
[메인] 오늘의 결정 (매일 7시)
  └ 30초 응답 → 1분 정체성 회복 화면 → 24h 후 결과 푸시

[보조 4-α] 이름 회복실 (주 1회 30분)
  └ 결혼 전 이름·전공으로만, 자녀·남편·시댁 얘기 금지

[보조 4-β] 내일의 나에게 추천
  └ 어제 후회한 사람 → 오늘 같은 결정 할 사람에게 메모
```

## 30~40대 엄마의 숨은 결핍 (UX·UI 설계 시 표면 욕구가 아니라 *이걸* 봐라)

1. 사라진 1인칭 — "○○맘"으로만 호명되는 정체성 소실
2. 판단 검증 결핍 — 매일 수십 개 결정, 검증할 곳 없음
3. 시간 빈곤·인지 과부하 — 5분 단위 멀티태스킹, 1분 이상의 흐름이면 이탈
4. 죄책감 화폐 — 모든 결정에 죄책감
5. 수평 동료 부재 — 새 친구 사귀기 너무 어려움

## 시간·마찰 제약

- 한 사용자가 앱에서 *연속 머무는 시간*은 최대 1~3분으로 가정
- 30초 응답이 메인. 그 이상 요구하면 이탈
- 푸시는 1일 1회 고정 7시 (강요 느낌 회피)
